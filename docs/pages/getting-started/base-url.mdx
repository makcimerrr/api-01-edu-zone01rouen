# Base URL & versioning

## Environnements

| Environnement      | URL                                              |
| ------------------ | ------------------------------------------------ |
| **Production**     | `https://api-zone01-rouen.deno.dev`              |
| Domaine custom     | (à confirmer auprès du mainteneur)               |
| Local              | `http://localhost:8000`                          |

> Tous les exemples curl de la doc utilisent la variable `$API`. Définis-la
> une fois pour copier-coller librement :
>
> ```bash
> export API="https://api-zone01-rouen.deno.dev"
> ```

## Préfixe de version

Tous les endpoints sont préfixés par `API_BASE_PATH` puis `API_VERSION`,
définis dans `config/config.ts` :

```ts
export const API_VERSION = "v1";
export const API_BASE_PATH = "api";
```

Soit en pratique :

```
$API/api/v1/<resource>
```

## Versions

| Version | Statut    | Notes                          |

| `v1`    | ✅ Stable | Voir [Versioning](/getting-started/versioning) |

## Health check

`GET /` redirige (`302`) vers `/docs/`. Pour un check programmatique,
utiliser n'importe quel endpoint léger :

```bash
curl -fsS $API/api/v1/promotions > /dev/null && echo OK
```
